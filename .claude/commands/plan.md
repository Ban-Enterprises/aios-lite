# Plan

> Plan something before you start building. A good plan saves hours.

## Variables

request: $ARGUMENTS

---

## Instructions

You're creating a PLAN, not executing it. Research, think, write it down.

Save plan to `plans/YYYY-MM-DD-{name}.md`

---

## How to plan

1. Read CLAUDE.md and relevant files from .context/
2. Understand what already exists and what needs to change
3. Ask the user if something is unclear

---

## Plan format

```markdown
# Plan: {title}

**Date:** YYYY-MM-DD
**Status:** Draft
**Goal:** {what we want to achieve}

---

## What we're doing
{2-3 sentences}

## Why
{How this connects to the user's priorities}

## Current state
{What already exists, what's missing}

## Steps

### Step 1: {Title}
{Description + specific actions}

### Step 2: {Title}
[...]

## Checklist
- [ ] {success criterion 1}
- [ ] {success criterion 2}

## Open questions
{If any — what needs to be clarified}
```

---

## After saving

Print:
1. Path to the plan
2. 2-sentence summary
3. "Ready to execute? I can start implementing."
